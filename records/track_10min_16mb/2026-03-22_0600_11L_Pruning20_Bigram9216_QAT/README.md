# 11L + Pruning 20% + BigramHash 9216 + QAT

## 목적

Exp 26 (11L + Pruning 10%)에서 val_bpb 1.13690 (SOTA 대비 -0.0055)을 달성했으나 크기 17.1MB로 1.1MB 초과. **크기를 16MB 이내로 줄이면서 val_bpb 우위를 유지**하는 것이 목표.

## 구현 방향

- **베이스**: Exp 26 (11L, pruning 10%, BigramHash 10240, val_bpb 1.13690)
- **핵심 변경**:
  - Pruning: 10% → **20%** (예상 절약 ~700-900KB)
  - BigramHash: 10240 → **9216** (예상 절약 ~100-150KB)
  - 합계: ~800-1050KB 절약 → 17.1MB - 1.0MB = ~16.1MB (빠듯)
- **그 외 설정**: 11L, int5 MLP + int6 Attn QAT(STE), FP16_KEEP blocks.9.attn.c_k, cosine warmdown 3500, SWA 0.35, eval stride=32

## 크기 vs 품질 트레이드오프

| 변경 | 크기 절약 | val_bpb 손해 |
|------|----------|-------------|
| Pruning 10%→20% | ~700-900KB | ~0.001-0.002 |
| BigramHash 10240→9216 | ~100-150KB | ~0.0005-0.001 |
| **합계** | **~800-1050KB** | **~0.0015-0.003** |

## 예상 결과

- **val_bpb**: ~1.138~1.140 (Exp 26 1.1369 대비 +0.001~0.003, 여전히 SOTA 1.1424보다 우수)
- **아티팩트 크기**: ~16.0-16.2MB (빠듯, 초과 가능성 있음)
- **학습 시간**: step_avg ~94ms (동일)
- **위험**: 중
  - Pruning 20%가 val_bpb에 미치는 영향이 예상보다 클 수 있음
  - 크기가 여전히 초과할 경우 추가 조정 필요
